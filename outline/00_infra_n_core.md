  ## Infra / Core
- **Rationale**
        
    DAW에는 두 가지 작업이 공존 :
    
    - 소리를 만드는 작업 : 오디오 드라이버의 버퍼 요청, 소리 출력
    - 나머지 모든 작업 : 화면을 그리고, 파일을 저장하고, 플러그인 로드하고, 사용자 입력 처리 등.
    
    스레드를 분리하여 작업을 처리. 
        
- **Audio Thread**
    - 소리 생성, 출력
- **Message Thread**
    - UI 이벤트 처리 (버튼, 슬라이더, 마우스)
    - 엔진에 명령 전달 (재생, 정지, BPM 변경)
    - 패턴 데이터 편집
    - UI 갱신
- **Background Threads**
    - 캐시로더 : 송 모드에서 재생 버튼을 누르면, 타임라인에 등장하는 모든 패턴의 모든 채널 샘플 파일을 미리 메모리에 디코딩.  오디오 스레드는 메모리에 있는 버퍼만 읽음. 파일은 재생 중 안건드림. 재생 중에 파일을 읽으면 디스크 IO가 오디오 스레드를 블로킹해서 소리가 끊기게 됨.
    - 오디오 녹음 데이터 디스크 쓰기 : 녹음 시작 → 종료
    - 플러그인 디렉터리 탐색 : 플러그인 스캔 → 완료
    - 하드웨어 MIDI 신호 수신 : 외부 입력장치 연결
    
- **1. Threads transmission : 스레드 간 통신**
    - 메세지 스레드 → 오디오 스레드 :  크고 복잡한 데이터는 더블 버퍼로 전달. 버퍼를 두 벌 두고, 메세지 스레드는 쓰지 않는 쪽에 변경 사항을 기록한 뒤 atomic 인덱스만 바꾸도록. 오디오 스레드는 인덱스가 가리키는 쪽을 읽도록.  단일 숫자는 std::atomic으로 직접 공유
    - 오디오 스레드 → 메세지 스레드 : 오디오 스레드는 메세지 스레드를 직접 호출 못하도록. 큐에 이벤트를 쌓아두면 메세지 스레드의 타이머가 매 틱마다 drain
    - 백그라운드 스레드 → 오디오 스레드 : 캐시로더가 디코딩을 마치면 atomic 인덱스를 스왑. 오디오 스레드는 다음 콜백부터 새 캐시를 읽도록.
- **2. Safety Rules & prevention of race condition**
    
    쓰레드가 여러 개일 때, 같은 데이터를 동시에 건드리는 상황이 있을 수 있는데, 누가 먼저 데이터에 접근하냐에 따라 결과가 달라짐. 따라서 순서를 강제해야 하는 경우가 생김.
    
    따라서 아래 세 가지를 상황에 맞게 사용:
    
    **더블 버퍼 + atomic 인덱스** — 덩어리가 큰 데이터. 두 스레드가 같은 메모리에 동시에 접근하지 않도록.
    
    **std::atomic** — 단일 숫자 하나. CPU 수준에서 읽기/쓰기가 쪼개지지 않음을 보장.
    
    **ScopedTryLock** — 락을 완전히 피할 수 없는 경우(오디오 클립, 플러그인). 오디오 스레드는 락을 못 잡으면 해당 블록을 그냥 건너뛰도록. 블로킹하지 않음.
    
    오디오 스레드 주의사항:
    
    - 메모리 할당 금지 (new, malloc, std::vector::push_back 등)
    
    - 뮤텍스 락 금지 (블로킹 가능성)
    
    - 디스크/네트워크 I/O 금지
    
    - OS를 부르는 모든 작업 금지
    
- **3. Memory Allocation**
    
    오디오 스레드는 콜백 안에서 메모리를 할당하지 않는 것이 원칙. 필요한 버퍼는 audioDeviceAboutToStart에서 미리 할당.
    
    stagingBuf              — 채널별 렌더링용 임시 버퍼 (2ch × bufferSize)
    mixerTrackBufs[8]       — 믹서 트랙 버스 버퍼 (8개 × 2ch × bufferSize)
    instrumentMidiBuffers   — 플러그인용 MIDI 버퍼 (16채널 × 2048 bytes)
    
    플러그인 처리 시 오디오 버퍼는 AudioBuffer<float>(ptrs, 2, safe)로 포인터 뷰만 만듬. 실제 할당이 아니라 기존 메모리를 가리키는 것.
    
    백그라운드 스레드는 자유롭게 할당 :
    CacheLoader는 모든 샘플 파일을 디코딩해 shared_ptr<AudioBuffer<float>>로 힙에 올리도록. AudioRecorder는 스레드 시작 시 임시 버퍼를 한 번 할당한 뒤 루프 안에서 재사용.
