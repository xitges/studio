## Audio Engine
오디오 드라이버의 버퍼 요청에 응답해 소리를 만드는 부분.
- **sound sources**<br>
음원 3가지 : 
  - Sample : wav/aiff 파일을 메모리에 올려두고 재생. 여러 개 보이스 슬롯 할당해서 동시에 여러 소리 낼 수 있도록. 새로운 연주 신호 들어올 때마다 슬롯 돌아가면서 할당해서 이전에 재생되던 소리가 끊기지 않고 겹쳐질 수 있도록.
    그대로 재생한다면 메모리에서 순서대로 그대로 읽어서 버퍼에 넣도록 구현. 피치 변환이 있다면 파일을 읽을 때, 읽는 크기를 늘리는데(1씩 읽는걸 1.05씩 읽는 등), 인덱스는 정수만 있으니까 주변 값들을 보간해서 버퍼에 넣도록 구현.
  - 내장 신디사이저 : 파형 그리고 그린 파형을 토대로 버퍼에 직접 값 넣도록. 보이스 슬롯 샘플과 동일하게. Oscillator -> filter -> ADSR envelop -> LFO 구조로 구현. 
  - 플러그인 : 고민이 필요함

- **signal pipeline**<br>
오디오 드라이버가 버퍼를 요청할 때 처리 순서 : <br>
버퍼 초기화 →  <br> 
  - if pattern 모드? 시퀸서가 bpm 기반으로 스텝 진행 → 스텝 발동 → 채널별 음원 트리거 noteEvent 검출 → 스레드 간 통신 → 보이스 재생 
  - if song 모드? 타임라인 위치 기준으로 클립 검출. 패턴 클립 → 내장 패턴 데이터로 음원 생성. audio 클립 → 디코딩된 버퍼에서 샘플 스트리밍 <Br><br>
→ 큐에 쌓인 SamplePlayer트리거 일괄 처리 →  믹서 FX 적용 →  마스터 출력 
- **voice scheduling**<br>
음원 트리거 신호 들어온 시점이랑 실제 보이스 생성되는 시점 스케줄링
  - 기본 스케줄링 규칙 :
    - 트리거 신호 → 재생에 필요한 정보 큐에 적재 → 버퍼 처리가 끝날 때 큐 순회하며 각 트리거마다 슬롯 할당 → 지정된 샘플 오프셋 위치에서 재생 시작
    - 내장 신디랑 플러그인은 큐 거치지 않도록.
